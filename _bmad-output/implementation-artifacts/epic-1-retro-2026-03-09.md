- **Epic status still "in-progress"** in sprint-status despite all stories being done
- **Undocumented port change** — Flask moved from planned port 5000 to actual 5001 without documentation
- **No test results documented** anywhere across any story

## Key Insights

1. Domain expertise matters — clustering was technically correct but financially naive
2. Documentation debt compounds fast — by the time we noticed, 4 epics had passed with empty records
3. Hybrid AI approach (curated list + Claude classification) is stronger than pure keyword matching or pure AI
4. Three-tier article routing (Watch List / Market Signals / not surfaced) adds a needed quality gate
5. Story specs with code templates and SQL patterns significantly accelerate development

## Significant Discovery: Theme Overhaul Required

### Problem

The current `THEME_DEFINITIONS` in `theme_clustering.py` uses 15 generic category-style themes. Asset managers organize their thinking around specific, named macro narratives — not broad categories.

### Decision: Option C — Hybrid Approach

- Curated Watch List of 21 specific narrative themes
- Claude-powered article classification with keyword fallback
- Three-tier article routing:
  - **Watch List** — article matches a narrative theme (theme_id assigned)
  - **Market Signals** — no theme match but has asset class tags (theme_id NULL, asset_tags present)
  - **Not surfaced** — no theme match, no asset class relevance (stored but hidden)
